buildDify 平台文档

对标 Dify 能力模型、从零构建的定制化 LLM 应用开发平台。一周冲刺产出可运行 MVP,贯通四条前后端可用链路。

✓ D1–D7 全部落地 💬 对话 📚 知识库 RAG 🔀 工作流 🤖 Agent 后端 55 tests passed

1 项目概览

四大能力,每一条都是贯穿前端 → API → 业务 → 数据 → LLM 的完整可演示链路。

💬对话应用 Chatbot

与模型流式(SSE)对话,会话与消息历史落库,自动建标题,可选绑定知识库。

📚知识库 RAG

上传文档 → Celery 分块 → embedding → pgvector 余弦检索 → 拼接上下文,回答带可溯源引用。

🔀可视化工作流

React Flow 编排;拓扑排序执行引擎 + 变量池 + 条件分支,节点级运行回放(输入/输出/耗时)。

🤖Agent + 工具调用

function-calling ReAct 循环;内置知识检索 / HTTP / 代码执行,思考轨迹 SSE 实时可视化。

设计铁律 · 请求严格单向:api → services → repositories → models。路由层不写业务、service 不写裸 SQL、LLM 调用一律走 app/llm 抽象,不在业务里硬编码厂商 SDK。

2 技术栈

关系数据与向量同库(pgvector),少运维一个服务;前后端各自现代工具链。

路由层 业务 / LLM 引擎 / 异步 基础设施 虚线 = 入队异步

9 Agent · function-calling ReAct

模型自主决定调用哪个内置工具,执行结果回灌进上下文,迭代至产出最终答复或达 AGENT_MAX_ITERATIONS(默认 6)。每步落 agent_thought,SSE 实时推送轨迹。

用户输入
→
provider.chat带工具规格
⇄
执行内置工具
观测结果回灌
→
最终答复

内置工具

📚knowledge_retrieval

复用 RAG 检索;dataset_id 可单配,缺省回退应用绑定库。

🌐http_request

GET/POST,仅 http(s);可配 allow_url_prefix 与响应截断。

🧮code_exec

受限 builtins 执行 Python,取 result 变量。

🧩可扩展

注册表加一条 BuiltinTool 即新增一种工具。

实测 · agent 应用启用 code_exec,提问「精确计算 (1234×567)+89」→ 模型自主调用 code_exec → 观测 699767 → 答 699767(=699767 ✓),轨迹落库可回放。

10 可观测

结构化请求日志

中间件为每请求生成 X-Request-ID,输出一行 JSON(method/path/status/耗时),并兜底未捕获异常(记堆栈 + 统一 500)。

token 计量与指标

GET /api/metrics 进程内请求数/错误数/平均时延/状态码分布;GET /api/metrics/usage 按模型汇总当前用户 token 用量。

11 快速开始

方式一 · 一键全栈(Docker Compose)

# 填入 LLM 密钥(见 backend/.env.example)
cp backend/.env.example backend/.env
docker compose up -d --build        # db / redis / api / worker / web

前端 localhost:3000 · API 文档 localhost:8000/docs。api 容器启动时自动 alembic upgrade head。

方式二 · 本地开发(热重载)

docker compose up -d db redis       # 只起依赖(db 映射宿主机 5433)

# 后端 backend/
uv sync && uv run alembic upgrade head
uv run uvicorn app.main:app --reload
uv run celery -A app.tasks.celery_app worker -l info   # 另开终端

# 前端 web/
pnpm install && pnpm dev
LLM 配置 · 至少配一个上游。实测讯飞 MaaS(OpenAI 兼容 + 768 维 embedding):LLM_PROVIDER=openai + OPENAI_API_KEY + OPENAI_BASE_URL + OPENAI_DEFAULT_MODEL + EMBEDDING_MODEL。或官方 Claude:LLM_PROVIDER=anthropic + ANTHROPIC_API_KEY。

12 四能力上手

登录后顶部导航在对话 / 知识库 / 应用 / 工作流四模块间切换。

  1. 注册登录

    首屏登录页注册账号 → 登录,token 存本地;401 自动清除并跳登录。

  2. 对话

    进对话页直接发消息,SSE 流式回复,刷新后会话列表与历史均在。右上可选知识库做 RAG。

  3. 知识库 RAG

    知识库页建库 → 上传 MD/PDF/TXT → worker 处理至 ready → 回对话选库提问,回答带 [n] 引用、可折叠溯源。

  4. Chatbot 应用

    应用页选 Chatbot 新建 → 配 prompt/模型/温度/绑定知识库 → 保存为新版本 → 发布 → 调试窗试聊 → 生成 API Key 经 POST /v1/apps/{id}/chat 对外调用。

  5. 工作流

    工作流页新建 → React Flow 拖搭「输入→检索→LLM→输出」→ 运行 → 节点全绿,逐节点展开看 resolved 输入 / 输出 / token。

  6. Agent 应用

    应用页选 Agent 新建 → 从工具目录添加 code_exec / http_request / knowledge_retrieval 并启用 → 调试窗提问,实时看到 💭思考 / 🔧工具调用 / 📥观测 轨迹与最终答复。
